# 代码审查清单

> 本清单适用于 Hermes 及所有 Agent 在提交代码前进行自我审查。  
> 来源：[[05-Knowledge/Reflections/hermes-coding-blindspots-20260415|Hermes 的代码失误复盘]]

## 输入处理与安全

- [ ] **不信任任何外部输入**：文件内容、用户输入、网络响应、环境变量
- [ ] **路径拼接前消毒**：拦截 `..`、绝对路径、空字符串、特殊字符
- [ ] **resolve() + 边界检查**：拼接路径后 `.resolve()`，再校验是否在允许目录内
- [ ] **文件大小上限**：读取前检查 `st_size`，超大文件跳过并告警
- [ ] **网络请求超时**：所有 HTTP/API 调用必须设 `timeout`

## 解析与标准库

- [ ] **标准库优先**：YAML 用 `PyYAML`，JSON 用 `json`，CSV 用 `csv.DictReader`
- [ ] **手写解析器必须有 fallback 注释**：明确标注「不支持 XXX」
- [ ] **测试边界 case**：写解析器前先列出 10 个异常输入

## 指标与逻辑完整性

- [ ] **多数据源验证**：定义指标前，列出所有可能留下痕迹的位置
- [ ] **活跃度检查至少覆盖**：内容页 frontmatter + log 记录 + audit 文件

## 文档-代码-模板一致性

- [ ] **三位一体检查**：修改 frontmatter/配置/接口时，同步检查「模板 → 示例 → lint 规则」
- [ ] **反向 grep 验证**：改完代码后，搜索所有相关示例和模板

## 可移植性与隐私

- [ ] **禁止硬编码本地绝对路径**：使用环境变量或相对路径
- [ ] **文档示例使用 `$VAULT` 或占位符**：避免泄露用户名和目录结构
